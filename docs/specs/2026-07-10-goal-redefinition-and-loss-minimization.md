## 1. なぜ方針転換するか（探索の総括）

2026-07-08〜10にかけて、「市場（1番人気）を単勝ROIで超える」という当初目標に対し、質的に異なる **10種類の独立した検証**を実施した。全て事前登録・Rule 3遵守・fold2完全OOSで測定している。

| # | 検証 | 結果 |
|---|---|---|
| 1〜8 | L1特徴量拡張（v40〜v50）、市場情報混入実験、var2.0.0 init_scoreトリック再現、JRAマイニングαゲート、複勝直接予測、Stern較正改善（λ再フィット・isotonic）、ワイド/馬連argmax EV選択 | 全て失敗（詳細: `docs/2026-07-09-current-system-guide.md` §6） |
| P1 | セグメント条件付きα異質性検定（5セグメント事前登録） | 確定4セグメント全てα=0、LRT p≈1.0。市場効率性はセグメント横断で成立 |
| P2 | 調教データTrack B（5候補、γのLRT） | 5候補全滅（p: 0.665/0.064/0.034/1.0/1.0、B-2/B-3はp<0.05だが事前登録閾値0.01未達） |
| P4 | 券種間整合性の系統的乖離測定（29セグメント） | 複勝×1番人気×多頭数で統計的に頑健な乖離を発見（TEST再現、D_adj+3.55pp、p≈0）が、ROI 84-86%で控除率の壁の内側に留まり収益化不可 |

**確定した結論**: 現行のJV-Link公式データ・4層アーキテクチャの範囲では、単勝・複勝・ワイド・馬連いずれの券種でも、市場を上回るROI（黒字化）を達成する経路は見つからなかった。P4は「ノイズではない本物の構造」を唯一発見したが、それでも控除率（約20-25%）の壁を超えられなかった。

**唯一の一貫したポジティブシグナル**: モデルの予測1位に無条件で単勝ベットした場合、ROIは市場（1番人気）よりわずかに高い。

| | 的中率 | ROI |
|---|---|---|
| モデル予測1位 | 29.99% | **81.89%** |
| 1番人気（市場） | 32.90% | 77.89% |

的中率は市場に劣るが、当てた時のオッズがやや高い（過剰人気馬への集中投票を避けている）ため、**損失率で見れば市場に対して+4.0pp優位**。これは`docs/2026-07-09-current-system-guide.md`にも記録済みの既知の結果であり、新たな検証ではない。

## 2. 新しい目標

### 2.1 主指標の変更

| | 旧目標 | 新目標 |
|---|---|---|
| 主指標 | ROI>100%（黒字化） | **1番人気ベースラインに対するROI優位**（損失最小化） |
| 合否基準 | L3 walk-forward ROI>100%（n≥200/fold） | L3 walk-forward ROIが1番人気ベースラインROIを統計的有意に上回ること（n≥200/fold） |
| L1/L2目標 | Top-1>30%、統合Top-1≥33% | 参考指標として保持（Phase合否には使わない） |

### 2.2 なぜこれが正当な目標か

- 控除率がある以上、**確率的に公正な賭けを続けても長期的には必ず損をする**（数学的事実、パラメータ調整で覆らないことは今回のセッションで実証済み）。
- したがって「黒字化」を唯一の成功基準にすると、達成不可能な目標に固執し続けることになる。
- 一方、「同じ損をするなら、最も損の少ない賭け方をする」は数学的に意味のある最適化問題であり、実際にモデルは市場（1番人気）に対して優位性を持つことが実測されている。
- 娯楽として馬券を購入する行為自体を禁止・推奨するものではなく、**購入するなら市場に blindly 従うより本モデルに従う方が期待損失が小さい**、という条件付きの結論である。

### 2.3 遵守事項（変更なし）

市場情報境界（L1市場情報禁止）、Rule 3（後出しじゃんけん禁止）、リーク停止閾値（Top-1>40%）、fold2 OOSプロトコルなど、他の全ての プロジェクト憲法条項は変更しない。損失最小化運用も同じOOS規律の上で評価する。

## 3. 損失最小化運用の実装スコープ

### 3.1 対象

`main/`（L4当日パイプライン）を復旧し、**実際に日次の推奨を出力できる状態にする**。

### 3.2 L3ベッティングロジックの変更

現行の`betting/src/backtest.py::simulate_bets`はEV閾値（`ev_threshold=1.05`）でフィルタする設計だが、これは「正のEVが存在する」ことを前提にしている。今回の結論（正のEVは存在しない）と整合させるため、日次運用では以下のロジックに切り替える:

- **選定**: 各レースでモデル予測1位（`pure_score_z`最大の馬）を単勝で選ぶ（EV閾値フィルタではなく、無条件選定）。
- **除外条件は維持**: `min_odds`〜`max_odds`（2.0〜50.0倍）、リスク上限（`max_daily_exposure`, `monthly_mdd_limit`, `consecutive_loss_stop`）は既存`betting_config.json`のまま使う（過剰な大穴・連敗による破滅リスクを抑えるため）。
- **サイジング**: 分数ケリーではなく、**固定額または固定比率**（例: 1レースあたりbankrollの0.5-1%）に変更する。ケリー基準は「真の確率で公正な賭け」を前提にした最適化であり、本戦略は「期待値が負であることを承知の上で損失を最小化する」という異なる目的のため、ケリーの数学的前提が成立しない。プランナーが具体的な固定比率を設計すること。
- **出力ラベル**: 生成される推奨CSV・ログ等に「本推奨は市場に対する相対的な損失最小化を目的とし、黒字化を保証するものではない（fold2 OOS実測: ROI 81.89%、元本の約18%の期待損失）」という定型の注記を必ず含めること。

### 3.3 L4パイプライン復旧範囲

- `main/unified_pipeline.py::run_unified_today()` を実データ（`main/data/race/race_ra.csv`, `race_se.csv`）で実行し、エラーなく最後まで到達することを確認する。
- 2026-07-09セッションで`main/build_today_features.py`をアーカイブ済み・`main/race_id_utils.py`切り出し済みのため、この経路が正しく動作するか要確認。
- `main/main.py`, `main/pipeline/strategy_pipeline.py`等、依然アーカイブ済みモジュールを参照している未使用コードは、実運用パスから到達しないことを確認した上で**削除または明確に隔離**する（refactorer相当の作業）。
- 当日オッズ取得（`main.jv_subprocess` / `common.data.src.get_data`経由のリアルタイムO1/O2/O3取得）が現在の実装で機能するか確認する。機能しない場合は、少なくとも「機能しない」ことを明記し、動作する部分（L1予測のみ等）に運用範囲を絞る。

### 3.4 検証基準

- 少なくとも1回、実際の当日データ（過去の実データで代用可）で`run_unified_today()`を通し、出力CSV（推奨・予測）が仕様通りの形式で生成されることを確認する。
- L3の新ロジック（固定比率サイジング）をfold2 OOSのTEST期間で再バックテストし、1番人気ベースラインに対するROI優位が実際に再現するかを確認する（Rule 3: 新ロジックのパラメータもVALID期間で決め、TESTは1回のみ）。
- 市場情報境界・リーク停止閾値など既存の全ゲートを引き続き適用する。
