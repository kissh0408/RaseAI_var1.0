## 1. 背景と制約

### 1.1 確定した中心的事実（全て実測値）

| 事実 | 数値 | 出典 |
|------|------|------|
| fold2 OOS で α=0（LRT p≈1.0） | L1 スコアは市場オッズに完全織り込み済み | `evaluation/reports/fusion_oos_fold2.json` |
| 市場 Top-1 / モデル Top-1 | 32.90% / 30.24%（-2.66pp） | `gate_summary.json` |
| 統合 logloss = 市場 logloss | 1.93124 / 1.93124（同一） | 同上 |
| 予測1位 単勝ROI | モデル 82.05% vs 1番人気 77.92%（+4.1pp） | alpha-gate 計画書 §0 |
| 予測1位 複勝ROI | モデル 84.63% vs 1番人気 85.15%（-0.52pp、Track C 判定 FAIL 済み） | `gate_summary.json` track_c_place |
| 単勝の正 EV ベット | OOS で 0 件（VALID 閾値選定も不成立） | 同上 phase3_l3_gates |
| ワイド argmax EV | 見かけ ROI 303% の 98.65% が 1 的中（14,257倍）に依存 → 集中度ゲートで不合格 | 指南書 §4.3 |

8 種類の独立した手法（指南書 §6）が全て同一の壁（α=0）に到達した。共通点は「**単勝勝率 p_win の全レース平均的な推定精度を上げようとした**」ことである。したがって次の一手は、この軸そのものを外れる必要がある。

### 1.2 遵守する制約（プロジェクト憲法）

1. **L1 に市場情報（オッズ・人気・`market_log_odds`・`init_score`）を一切入れない**。市場情報が正当に使えるのは L2（統合変数）・L3（EV/サイズ）のみ。
2. **Rule 3（後出しじゃんけん禁止）**: 閾値・セグメント・パラメータの選定は fit/VALID 期間（〜2024）のみで行い、TEST(2025+) は各候補 1 回だけ。
3. **リーク停止閾値**: Top-1 > 40% または Spearman > 0.6 → 即停止・evaluator へ報告。
4. **αゲート新プロトコル**（A-3、CLAUDE.md 反映済み方針）: 一次審査は Top-1 ではなく γ の LRT p<0.01（2023 fit → 2024 eval）。Top-1 は退行防止条件（fold2 基準 29.99% 以上）に格下げ。
5. **正式判定は fold2 OOS のみ**。15 モデル全 fold 平均は contaminated として不使用。

## 2. 却下する方向性とその理由

以下は「§6 の 8 手法の変奏」であり、同じ α=0 の壁に当たることが構造的に予見されるため**原則却下**する。

| 却下する方向 | 該当する §6 手法 | 却下理由 |
|---|---|---|
| L1 特徴量の水準系追加（成績・条件・血統・騎手の集計値の拡充） | #1（v40〜v50 全滅）、#2 | 全レース平均の p_win 推定精度を狙う軸。市場は既にこれを織り込んでいる（α=0）。 |
| 市場情報の混入・残差学習（init_score 等） | #2, #3 | 真の OOS で 27.64% まで悪化した実証済み失敗。憲法違反でもある。 |
| 確率算出式の変更・再較正（Stern λ 再フィット、isotonic、Harville 切替の再検討） | #6, #7 | 点推定改善が誤差の範囲内（ブートストラップ CI がゼロを含む）と確認済み。入力情報が同じなら出力形式を変えても情報量は増えない。 |
| **1着・2着・3着の個別直接予測**（§7 未着手領域の一つ） | #5 の変奏 | 複勝(top3)直接予測が Stern 逆算に logloss で有意に劣後した実証がある。目的変数を変えても入力特徴量と情報源が同一である限り、質的に同じアプローチ。**却下**。 |
| ワイド/馬連の EV 選択方式の工夫（argmax 以外の選択規則探索） | #8 の変奏 | 選択規則を変えても、確率が市場確率と同一（α=0）である限り正の EV は選べない。winner's curse の再演リスクが高い。 |
| 複勝市場での EV フィルタ探索の継続 | Track C 測定済み | 確定払戻ベース（モデルに最も有利な上限診断）ですら ROI 84.63% < 100%、かつ 1 番人気に -0.52pp 劣後。不可能性テストが FAIL しており、事前オッズではさらに悪化する。**却下**（測定完了として記録済み）。 |

| 領域 | 採否 | 理由 |
|---|---|---|
| 調教データ深掘り（Track B） | **採用**（提案 P2） | 事前登録済み・ユーザー承認済み・αゲート基盤稼働済み。「水準」ではなく「変化・頻度・形状」軸で、判定軸も Top-1 ではなく γ（市場条件付き尤度）に変更済み。§6 #1 の変奏との差分は §3 P2(d) に明記。 |

### P1: セグメント条件付き α の異質性検定（市場が構造的に弱い区分の事前登録探索）

**内容**: α=0 は「全レース平均で L1 が市場に情報を足さない」という主張であり、「あらゆる部分集合で足さない」ことは意味しない。市場の情報集約が構造的に弱いと事前に予想されるセグメントを**ドメイン知識のみから事前登録**し、各セグメント内で fold2 OOS スコアの条件付きロジット（α, β）を fit 期間（2023 fit → 2024 eval、αゲートと同一プロトコル）で再推定し、セグメント別 α>0 の LRT 検定を行う。

**事前登録セグメント案**（fit 期間データの件数確認後に確定。TEST は見ない）:

1. 新馬・未出走馬中心レース（過去走情報が市場にも乏しい ⇔ 調教・血統の相対価値が高い）
2. 少頭数レース（8頭以下。流動性が低く市場の集約が粗い）
3. 重・不良馬場（当日変化する条件。市場の反応が遅れる可能性）
4. ローカル開催場（福島・新潟・小倉等。中央場より投票参加者の情報水準が低い可能性）
5. 低クラス条件戦（注目度が低く市場参加者の分析投資が薄い）

- (a) **実装コスト**: 1〜2 セッション。`prob_fusion/src/fit_fusion.py` と `evaluation/alpha_gate.py` の既存 MLE をセグメントフィルタ付きで回すのみ。新規データ取得ゼロ。
- (b) **検証時間**: 数日以内。fit 期間内で完結（TEST は有意セグメントが出た場合のみ各 1 回）。
- (c) **成功時の期待効果**: あるセグメントで α>0 が確定すれば、そのセグメント限定で L2 統合→L3 EV が初めて意味を持つ。ベット対象が全体の 10〜20% に絞られても、正の EV が存在する初の領域になる。定量目安: 該当セグメントで logloss が市場を有意に下回ること（現状は全体で差ゼロ）。
- (d) **§6 との質的差分**: 8 手法は全て「**大域的な** α（または Top-1）」を動かそうとした。本提案はモデルにも確率式にも一切手を触れず、「**市場効率性の異質性**」という未検定の仮説を検定する。L1 の情報量を増やす試みではなく、既存情報が価値を持つ場所を探す試み。
- (e) **失敗メカニズム / 早期打ち切り**: (i) 多重検定による偽陽性 → セグメント数 K で Bonferroni 補正（p < 0.01/K）を事前登録。(ii) セグメント内 n 不足で検出力ゼロ → fit 期間で n<300 レースのセグメントは事前に検定対象から除外。**打ち切り基準**: 全登録セグメントで補正後 p≥0.01/K → 「市場効率性はセグメント横断的に成立」と記録し終了（1 サイクルで完結、延長・セグメント追加の後出しは禁止）。

### P2: Track B — 調教時系列 5 候補の αゲート審査（事前登録済み）

**内容**: `2026-07-08-alpha-gate-data-expansion-plan.md` Track B の B-1〜B-5（強度トレンド・頻度変化・加速度プロファイル変化・失速率トレンド・調教場切替）を 1 本ずつ αゲート（γ の LRT）にかける。HC 511万行（2015〜）・WC 74万行（2021〜）は取得済み。αゲート基盤は稼働済み（ネガティブコントロール γ=0 を正しく判定済み、`gate_summary.json` alpha_gate 節）。

- (a) **実装コスト**: 候補 1 本あたり約 1 セッション（特徴量生成 + merge_asof リーク防止 + ゲート実行）。5 本で 4〜6 セッション。
- (b) **検証時間**: 1〜2 週間（逐次審査）。
- (c) **成功時の期待効果**: γ 有意な候補は L2 の追加信号スロット（γ 項）として直接本番化できる（L1 再学習不要 — B-3 規約）。1 本でも通れば「市場条件付きで尤度を足す信号」の初の実例となり、ΔLL/race > 0 が EV の源泉になる。
- (d) **§6 との質的差分**: #1（v40〜v50）との差分は 2 点。(i) 情報源: 市場参加者に集約形でしか流通しない**個体内時系列の変化・頻度・形状**（v50 が失敗した「水準の正規化」は除外済み）。(ii) 判定軸: Top-1 ではなく **γ（市場条件付き尤度）**。Top-1 が不変でも γ 有意なら採用、逆も然り — §6 の 8 手法はいずれもこの審査軸で評価されていない（#4 のみ γ 審査だが、候補が pure_score_z と相関 0.73 の既知情報だった）。
- (e) **失敗メカニズム / 早期打ち切り**: 調教の時系列変化も厩舎関係者・専門紙経由で市場に織り込まれている場合、全候補 γ=0 になる。**打ち切り基準**: 計画書 B-4 のとおり 5 ファミリ全滅で「JV-Link 調教データからの市場超え信号は現行粒度では不成立」と記録し、候補の追加生成（後出し）はしない。各候補は実装前に 10 行の事前登録を `docs/specs/` に書く。

### P3: 真のレース前オッズ時系列 — 却下（2026-07-09 再検証済み）

**却下確定**。`docs/JV-Data.md` §6(2)速報系データを直接確認した結果:

- **時系列オッズ（単複枠）0B41 / 時系列オッズ（馬連）0B42** が実在し、レース毎の複数時間帯オッズを提供する（提供期間1年間、他の速報系より長い特別枠）。
- しかしこれらは「**速報系データ**」に分類され、リアルタイム配信された瞬間にしか捕捉できない。「**蓄積系データ**」（過去の任意の日付を後から再取得できる仕組み）には含まれない。
- したがって **2015〜2025年の既存データに遡って取得することは不可能**。取得できるのは「今から購読を開始した場合の、これから行われるレースの分」のみ。

これにより当初想定していた「Phase 0: 1セッションで調査完了 → 蓄積系にあれば安価に検証」というシナリオは成立しない。実際には:
1. 今すぐ購読を開始しても、意味のあるレース数（fit/eval に必要な最低サンプル）が貯まるまで数週間〜数ヶ月を要する。
2. 貯まったデータは既存の fold2 OOS 期間（TEST=2025-01-01〜）に統合できず、**新しい別のTEST期間**としてしか扱えない。過去の学習・検証フレームワークをそのまま使えない。
3. 「安い調査」ではなく「時間のかかる先行投資」であり、Part I の他提案（P1/P2/P4、いずれも既存データのみで完結）と比較したときのコスト構造が根本的に異なる。

**却下理由**: Rule 3 の「事前登録して安価に検証する」という設計原則にそぐわない。前向きデータ収集自体は技術的に可能なので、将来的にユーザーが「数ヶ月単位の投資をしてでも市場動態を研究したい」と判断した場合は独立した意思決定として再提案する（今回の優先順位付けからは除外）。

（旧内容: フィージビリティ調査とオッズドリフト研究の段階構成案は本却下により不要となったため削除）

### P4: 券種間整合性の系統的乖離測定（単勝由来確率 vs 複勝・ワイド・馬連の確定払戻）

**内容**: 単勝・複勝・ワイド・馬連は別々のプールで価格形成される。単勝オッズから Stern/Harville で逆算した理論確率と、他券種の確定払戻が示す実効確率の**系統的乖離**（例: 特定の人気帯・頭数帯で複勝が理論比で常に過小/過大評価される）を fit 期間で測定する。乖離が系統的なら、L1 を一切使わず「市場 vs 市場」の裁定的エッジとなる。既存資産（HR 払戻 2015〜2026、`pair_probs.py`、`place_prob.py`、較正測定基盤）だけで実施可能。

- (a) **実装コスト**: 1〜2 セッション（既存の較正誤差測定 — ワイド Stern 8.35pp、馬連 Harville 16.01pp — を「誤差の符号と条件依存性」の分析に拡張するのみ）。
- (b) **検証時間**: 1 週間以内（fit 期間で完結、TEST 1 回）。
- (c) **成功時の期待効果**: 系統的乖離が見つかれば、乖離方向にのみベットする戦略の ROI 上限を確定払戻で診断できる（不可能性テスト）。既測定の較正誤差（ワイド 8.35pp、馬連 28.65pp/16.01pp）は無視できない大きさであり、その一部が「ランダム誤差」ではなく「条件依存の系統誤差」である可能性は未検定。
- (d) **§6 との質的差分**: #8（argmax EV）は「モデル確率 p と市場オッズの差」を探した。本提案はモデル確率を使わず**市場内部の自己不整合**を探す。α=0 は「L1 vs 単勝市場」の命題であり、「単勝市場 vs 複勝市場」の整合性には何も言っていない。
- (e) **失敗メカニズム / 早期打ち切り**: (i) 乖離は控除率差とテイクレートで説明され尽くす可能性が高い（複勝の高控除率下で 15pp 超のミスプライスが必要）。(ii) 確定払戻は事前オッズではないため、乖離が見えても購入時点で消えている可能性（Track C と同じ限界）。**打ち切り基準**: fit 期間で「控除率調整後の系統乖離が最大人気帯でも 3pp 未満」なら ROI>100% は数学的に届かないので終了。winner's curse 対策として払戻集中度ゲート（`payout_not_concentrated_top1_lte_30pct`, `n_hits_gte_10`）を必ず適用。

## 4. 比較表と優先順位

| 提案 | コスト | 検証期間 | 新規データ | 成功確率の定性評価 | 成功時の価値 | Rule 3 リスク |
|---|---|---|---|---|---|---|
| P1 セグメント別α | 1〜2 セッション | 数日 | 不要 | 低〜中（市場は概ね効率的だが異質性は未検定） | 中〜大（初の正 EV 領域） | 低（Bonferroni + 事前登録で制御） |
| P2 調教 Track B | 4〜6 セッション | 1〜2 週間 | 不要（在庫済み） | 低〜中（市場の織り込み粒度に依存） | 大（γ 項として直接 EV 化） | 低（事前登録 5 本固定） |
| ~~P3 オッズ時系列~~ | — | — | — | **却下**（2026-07-09、速報系のみで蓄積系なし。過去データに遡及不可） | — | — |
| P4 券種間乖離 | 1〜2 セッション | 1 週間 | 不要 | 低（控除率の壁が厚い） | 中（L1 非依存のエッジ） | 中（多条件探索になりやすい → 条件を事前固定） |
| P5 L4 復旧 | 2〜3 セッション | 数日 | 不要 | —（性能向上でない） | 運用整備のみ | — |

**優先順位**: P1 → P2 → P4 → P5（P3 は却下・除外）

判断根拠:
- P1 は「期待値 × コスト」比が最良。既存インフラ（`fit_fusion.py` / `alpha_gate.py`）の再利用のみでコストが最小、かつ**結果がどちらに転んでも後続の指針になる**（有意セグメントが出れば P2 の探索をそこに集中でき、全滅なら「セグメント横断で市場は効率的」という強い事実が確定して撤退判断 §5 の材料になる）。
- P2 はユーザー承認済み・事前登録済みで確実に実施すべきだが、コストが P1 の 3 倍であり、P1 の結果（どのセグメントで調教情報が効きうるか）を先に知る方が審査の解像度が上がる。
- P3（レース前オッズ時系列）は速報系データのみで蓄積系に存在しないことが確定したため却下（§3 P3 参照）。前向き収集は数ヶ月の投資判断を要する別軸の意思決定であり、今回の優先順位から除外する。
- P4 は控除率の壁で成功確率が最も低く、Track C の FAIL（複勝上限 84.63%）が既に悲観的な傍証を与えている。

## 5. 推奨する次の一手（1 つ）

**P1: セグメント条件付き α の異質性検定** を推奨する。

**最初の検証ステップ**（実装は implementer に委譲）:

1. fit 期間（2023-01-01〜2024-12-31）で上記 5 セグメント案の該当レース数を集計し、n≥300 を満たすセグメントのみを検定対象として本書 §6 の事前登録表を確定する（**この時点で TEST には一切触れない**）。

## 7. 学習方法（Learning Methods）のパラダイムシフト

現在の L1（`pure_rank/`）は LightGBM LambdaRank を用い、各馬の特徴量を独立したレコードとして評価しランキング関数で順位付けする。しかし競馬は独立したタイムトライアルではなく、出走馬同士が道中のペース・位置取り・風圧の遮断などを通じて互いに影響し合う「動的な相互作用のネットワーク」である。この文脈を捉えるため、深層学習に基づくより高度なアーキテクチャへの移行が候補となる。

### 7.1 Transformer と Self-Attention 機構による相互作用のモデル化

出走馬のスタート位置（枠順）や相対的な関係性が事前に与えられているという構造は、自然言語処理における Sequence-to-Sequence 問題と数学的な類似性を持つ。従来は LSTM/GRU 等の RNN が系列データに用いられてきたが、レース中の馬同士の相互作用は一方向ではなく常に全体が影響し合うため、自己注意機構（Self-Attention）を備えた Transformer が適している。

Transformer の強みは、入力された出走馬リスト全体を一つの文脈として捉え、ある馬の存在が他の馬のパフォーマンスに与える影響を Attention の重みとして直接学習できる点にある。

- **JRA データのケーススタディ**: 12 種類の従来型機械学習モデル（決定木ベース・線形モデル等）と比較し、Transformer ベースのアーキテクチャは予測精度を維持しながら 111% を超える回収率を達成したとの報告がある。
- **香港ジョッキークラブ（HKJC）データの研究**: 予測タスクを「情報検索のランキング問題」として再定式化し、RankNet を高速化した LambdaRank の深層学習版損失関数（Ranking Loss）で Transformer を学習させるアプローチ。$m$ 頭の間に存在する $m\mathrm{C}2$ 通りのペア全てに対するバイナリクロスエントロピー損失を累積することで、着差の極めて小さいレースでも精緻なランキング学習が可能。1 レースあたり 1 ドルの賭けに対して平均 1.12 ドル（利益率 12%）、複利運用で約 7% の安定利益率と報告。
- **新馬戦での優位性**: 過去のパフォーマンスデータが乏しく、オッズメーカーや一般ベッターにとって予測困難な「新馬」のレースで、Transformer が卓越した予測能力を発揮したと報告されている。血統や調教パターンの隠れた相互作用を捉えた可能性が示唆される。

**本プロジェクトへの示唆**: 現行 LambdaRank も既にペアワイズ／リストワイズ損失（NDCG 最適化）でランキングを学習しており、Transformer の主張する優位性の核心は「Self-Attention によるレース内相互作用の明示的モデリング」にある。LightGBM は特徴量エンジニアリングで擬似的にこれを表現している（`field_` プレフィックスのレース内相対 Z スコア等）。真の優位性があるかは、**同一の fold2 OOS プロトコルで直接比較する以外に検証方法がない**。

### 7.2 Siamese Neural Network（双子型ネットワーク）による相対評価と距離計量

絶対的なタイムや単一の勝率を予測するのではなく、レース内の「馬と馬の相対的な力関係」を直接学習させるアーキテクチャ。全く同じ重みを共有する複数のネットワークに異なる入力ベクトル（異なる馬の特徴量）を同時に流し込み、出力される潜在ベクトル間の「距離」を比較する。学習目標は、類似オブジェクト間の距離計量を最小化し、異なるオブジェクト間の距離を最大化すること（Triplet Loss / Contrastive Loss）。

出走馬すべてのレーティング $R_h$ を同一パラメータセットで並列計算する関数 $R(x_h)$ として機能し、Softmax で予測確率に変換する:

$$\hat{p}_h = \frac{\exp(R_h)}{\sum_i \exp(R_i)}$$

最適化には「Win-Log-Loss（$L(\hat{p}_w) = -\log(\hat{p}_w)$）」を用いる。この構造により、ネットワークは勝馬の評価を上げるだけでなく、同じレースで敗北した馬たちの評価を同時に相対的に下げるように重みを更新する——レース競技のゼロサム的性質を単純な2値分類より正確に表現する。

日本の競馬データ（1954〜2017年、約10万レース）を用いた研究では、2頭間の「走破タイム差」をターゲットラベルとして学習し、そこから最終的な勝者を導く試みの有効性が示唆されている。

**本プロジェクトへの示唆**: これは概念的には LambdaRank のペアワイズ損失（`label_gain` による順位差の重み付け）と近い発想であり、「何が新規性か」は Softmax 正規化された確率出力を直接学習に組み込む点にある。現行アーキテクチャは L1（ランキングスコア）と L2（確率化）が分離しているが、Siamese 型は両者を一体で学習する。

### 7.3 HodgeRank と位相幾何学的データクレンジング

対戦型競技のノイズの一つが、過去の直接対決データにおける非推移的（Intransitive）・循環的な結果（「A は B に勝ち、B は C に勝ったが、C は A に勝つ」）。決定木ベースのアルゴリズムはこれに混乱しやすい。**HodgeRank**（トポロジーにインスパイアされたランキングアルゴリズム）は、ペアワイズ比較行列を「推移的で一貫性のある成分」と「非推移的で循環的な成分」に数学的に分解する。ケメニー最適化が NP困難であるのに対し、HodgeRank は線形最小二乗回帰で計算できる利点がある。

イギリスの競馬データ（2005〜2014年、70,261レース）を用いた研究では、馬を「0次元単体」、直接対決スコアを「1次元単体」とするフラグ複体表現でデータをモデル化し、HodgeRank で矛盾データを排除した結果、ホールドアウトサンプルで小さな利益を達成し、イギリス競馬市場の非効率性を示したと報告されている。

**本プロジェクトへの示唆**: これは新モデルというより**前処理・メタ特徴量**としての適用余地がある。既存の `hist_*` 系特徴量（過去走の相対成績集計）に対して、非推移的ノイズの除去を追加できるか検討可能。ただし JRA では香港ほど「同一馬同士の直接対決」が頻繁ではない（クラス・距離・馬場が毎回変わる）ため、有効なペアワイズ比較データの密度が英国の平地競馬より薄い可能性が高く、適用前にデータ密度の確認が必要。

### 8.1 グラフニューラルネットワーク（GNN）による関係性の抽出

### 8.3 Benter の残差理論と動的レーティングシステム

Bill Benter は、モデルの予測と実際のパフォーマンスの誤差（残差）を単なるノイズではなく「その馬が持つ未知の距離適性や馬場適性」の表れとして解釈した。過去の各レースの残差を、今日のレースの距離・条件との類似性に対する関係性として推定することで、出走回数が少なく統計的有意性が低い馬でも隠れた適性を数値化できる。

また、単純な「過去5走の勝率」に代わる動的指標として、Elo-MMR・TrueSkill・Glicko 等のレーティングシステムの導入も候補となる。対戦相手の強さを考慮して単一数値に能力をカプセル化し、ベッティング期間中は変動しない静的データとして機能する。香港の研究では、Transformer + Elo-MMR レーティングの組み合わせが最高精度（21.4%）を記録したと報告されている。

**本プロジェクトへの示唆**: 残差ベースの適性推定は、既存の `hist_best_time_same_cond`（距離条件別最良タイム）等と概念的に重なるが、「モデル予測との差分」を明示的な特徴量にする点が新しい。Elo 系レーティングは JRA のような「毎回対戦相手が大幅に変わる」競技でも実装しやすく、比較的低コストで試せる候補。ただしこれも §8.1 と同様、**「全レース平均の水準系特徴量」に分類されうるため、αゲート審査が前提**。

| 特徴量カテゴリ | 抽出元データ・手法 | 捉えることのできる非効率性・文脈 |
|---|---|---|
| グラフ特徴量 | 有向グラフ構築, Node2Vec, GNN | 騎手・調教師間の隠れた勝負パターンのネットワーク構造 |

## 9. 予測方法（Prediction Methods）と確率推論の再構築

現行 L2 の α=0 という結果は、機械学習スコアを市場オッズと単純な対数線形回帰（条件付きロジット）で統合するアプローチが、効率化されたパリミュチュエル市場ではもはや機能しないことを示している。

### 9.1 Benter の2段階条件付きロジットモデルの限界と非線形融合

Benter が香港市場で用いた核心技術は「2段階アプローチ」——第一段階でファンダメンタルモデルを推定し、第二段階で自前の予測確率と市場オッズの2変数のみを条件付きロジスティック回帰（clogit）で再統合する手法である。clogit はレースごとにデータをグループ化し、グループ内の確率合計が1になる制約を保ったまま依存関係をモデル化できる点が画期的だった。

しかし現在は市場オッズが過去の戦績・タイムをほぼ完全に反映しており（効率的市場仮説の成立）、現行システムのように全体に一定の α, β を適用する線形統合では、市場オッズの表現力（β=1.0343）にファンダメンタルモデルが飲み込まれる。

**代替案**: 多層パーセプトロン（MLP）による統合レイヤーで、市場オッズが過剰反応しやすい特定条件下（メディアで過剰に持ち上げられた馬、人気騎手騎乗による実力以上のオッズ低下等）でのみ、内部的にファンダメンタルモデルの比重（α に相当する Attention）を動的に引き上げる非線形統合。加えて、締切直前までのオッズ変動軌跡（Dynamics of Odds）自体を統合レイヤーの特徴量として組み込み、スマートマネーの流入と市場パニックを区別する。

**本プロジェクトへの示唆**: 「市場が過剰反応する条件」を条件付き（セグメント別）に検出するという発想は、**Part I の P1（セグメント条件付き α の異質性検定）と本質的に同じ仮説**を、線形の条件付きロジットではなく非線形（MLP）で検定する提案である。P1 で有意なセグメントが発見された場合、その拡張として MLP 化を検討する価値がある。オッズ変動軌跡の活用は **P3（レース前オッズ時系列）が却下確定（§3 P3 参照、速報系のみで過去データなし）のため、この経路も実行不可能**。

### 9.2 多頭数確率計算の進化: Harville/Sternから一般化Plackett-Luceへ

現行の連勝式（馬連・ワイド・3連複等）確率計算は Harville の公式と Stern の近似モデルに依存する。

- **Harville モデルの欠陥**: 「選択の公理（IIA: 無関係な選択肢からの独立性）」を前提とする——ある馬が1着になる確率と、その馬を除いた残りの馬の相対的力関係は不変という仮定。しかし極端に強い馬がいる場合、後方の2着争いのペースや展開が歪む。走破タイムが正規分布に従う場合、Harville モデルは「極端に強い馬」「極端に弱い馬」の確率評価に系統的バイアスを生じ、穴馬の2着確率を過小評価することが数学的に示されている。
- **Stern モデルの限界**: 走破タイムをガンマ分布としてモデル化し補正を試みるが、計算コストが高く、複雑な連勝式では最尤推定の近似誤差が累積しやすい。
- **代替案: 一般化 Plackett-Luce（PL）モデル**: ランキング確率を柔軟に計算する統計モデルで、近年 PyTorch 等の深層学習との統合が進んでいる。特徴量ベクトルから直接 PL モデルのパラメータを推論する「一般化 Plackett-Luce 分布」がノイズ除去スケジュール（Denoising Schedule）と組み合わせて提案されており、IIA の制約や極端オッズ時のバイアスを回避し、エキゾチックベット（3連単・3連複）の適正オッズを高精度に計算できるとされる。

**本プロジェクトへの示唆**: これは今回のセッションで実測した**ワイド/馬連の Stern vs Harville 較正誤差問題**（ワイド: Stern 8.35pp / Harville 13.55pp、馬連: Stern 28.65pp / Harville 16.01pp）に直結する。現行システムは券種ごとに「較正誤差が小さい方」を経験的に選んでいるが、PL モデルはこの二択自体を置き換える第三の選択肢となりうる。**ただし導入する場合も、既存の `betting/analysis/compare_pair_probability_models.py` と同一の fold2 OOS プロトコルで PL モデルの calibration_max_error_pp / logloss を測定し、Stern・Harville と直接比較することが必須**（Part I の P4 と同種の「市場内部の整合性」軸に近く、L1 の α=0 とは独立に検証可能）。

## 10. 購入戦略（Purchase Strategies）と数学的資金管理

L3 の現状の課題は「予測 EV 最大のものを選ぶ（argmax EV）」という単純なロジックに起因するバンクロール崩壊リスクである。これは本セッションで実測済み（ワイド ROI 303% の 98.65% が単一の14,257倍的中に依存、winner's curse）。

### 10.1 大穴・本命バイアス（FLB）の構造的搾取

競馬市場で半世紀以上確認されているアノマリーが Favorite-Longshot Bias——高オッズの大穴馬券が過剰に買われ、低オッズの本命馬が過小評価される現象。米国560万レースの分析では、オッズ100倍以上の大穴の平均回収率が約-61%、本命の回収率損失はわずか-5.5%と報告される。背景にはプロスペクト理論（低確率の過大評価）とオッズメーカーによる「戦略的欺瞞」（意図的な朝オッズの大穴インフレ）の2説がある。

**対策案**: モデルの EV に関わらず超高オッズ帯を無条件除外するハードフィルター。ある検証では上限オッズ閾値を「4.9倍」に設定した場合に分散を抑えつつ最も安定したプラスエッジを抽出できたと報告されている。

**本プロジェクトへの示唆**: 現行の `betting/config/betting_config.json` には既に `max_odds` パラメータがあり（単勝バックテストで運用中）、思想自体は導入済み。ワイド/馬連の `run_backtest_oos_pairs.py` には現状オッズ上限フィルターが未実装——**これは既存インフラへの小さな追加で即座に試せる、コストの低い改善候補**。ただし本プロジェクトの α=0 という結論下では、そもそも「勝てる母集団」が存在しない可能性が高く、フィルターだけでは正の EV は作り出せない点に注意（FLB フィルターは分散削減であって、エッジの創出ではない）。

### 10.2 勝者の呪い（Winner's Curse）と統計的縮小推定量（Shrinkage Estimator）

100以上の候補から argmax で選ばれた「最も EV が高い」選択肢は、真に能力が高いのではなく「モデルの推定誤差が最も上振れしただけ」の可能性が高い——これは本セッションで実測確認済みの現象そのものである。

**対策案**: 縮小推定量（Shrinkage Estimator）——予測 EV を額面通りに受け取らず、ベイズ推定の考え方で平均（事前期待値）方向へ縮小する。研究によれば、統計的に限界的な有意水準（Z値1.96相当）のシグナルは、真の信号対雑音比がその予測値の約1/2.47に過ぎないとされ、縮小係数での下方修正が推奨される。

**本プロジェクトへの示唆**: これは今回追加した `payout_not_concentrated_top1_lte_30pct` / `n_hits_gte_10` ゲート（事後検出）よりも一段進んだ**事前対策**にあたる。argmax 選択前に、候補ペアの推定確率 $p$ を件数の少ない（分散の大きい）候補ほど強く事前平均へ縮小してから EV を計算する設計は、次にワイド/馬連の argmax 選択ロジックを改修する際の具体的な実装候補になる。ただし縮小してもなお正の EV が残らない可能性が高い（母集団に真のエッジがないため）ことは事前に認識しておくべき。

### 10.3 マルチアウトカム・ケリー基準と KKT 最適化

同一レース内で複数の買い目（単勝・複勝・ワイド等）に資金をどう配分するかは、独立した1点買いの単純なケリー基準では解けない（相互排他性・相関により資金100%超のベット要求が生じうる）。**マルチバリエイト・ケリー基準**は凹構造を持つ最適化問題として、Karush-Kuhn-Tucker（KKT）条件を用いた凸最適化（逐次二次計画法 SQP、または Smoczynski のアクティブセットアルゴリズム）で解く。手元に残す現金を「すべての状態に対するポジション」と解釈し、有利な結果に対してのみ資金をトップアップする形で定式化される。香港ジョッキークラブのデータでの適用では、複利運用による資産曲線の安定化（平均リターン率1.07）に成功したと報告されている。

**本プロジェクトへの示唆**: 現行 `betting/src/kelly_sizer.py` は単勝のみの単純フラクショナル・ケリーであり、複数券種の同時最適化は未実装。**ただし、これは「正の EV を持つ買い目が複数存在する」ことが前提の技術であり、本プロジェクトは現状どの券種でも正の EV が確立していない（§1.1 参照）。したがって導入優先度は低い——エッジが1つでも確立してから着手すべき**（Part I の P5 と同じ「エッジ確立後」の位置づけ）。

### 10.4 リスク制約付きケリー（Risk-Constrained Kelly）とプールディプレッション回避

パリミュチュエル市場では自身のベット額がプールに流入しオッズ自体を押し下げる（Pool Depression）。完全なケリー基準は長期成長率を最大化する一方、短期的なドローダウンリスクが高い。

**対策案**:
- **フラクショナル・ケリー**: 算出ベット額に係数（0.25〜0.5）を乗じて縮小し、パラメータ推定誤差への安全マージンを確保。
- **リスク制約付きケリー（Busseti らの手法）**: 長期成長率最大化に対し「指定レベルのドローダウン発生確率を一定以下に抑える」制約を追加。

**本プロジェクトへの示唆**: `betting/config/betting_config.json` の `kelly_fraction=0.08` は既にフラクショナル・ケリーを採用済み（0.08 は保守的な値）。リスク制約付きケリーへの拡張は技術的には可能だが、これも §10.3 同様「エッジ確立後」に優先度がある。

## 11. Part II の結論とロードマップ（外部サーベイ原文の結論部）

現状システムの α=0 という性能頭打ちは、アプローチそのものの敗北ではなく、「表形式データの線形処理（L1）」「Harville に基づく不正確な確率計算と単純な線形統合（L2）」「勝者の呪いとポートフォリオ分散を無視した単一点買い（L3）」という旧来アーキテクチャの限界が同時に露呈した結果、という見立てが原文の主張である。

原文が提示する統合ロードマップ:

1. **L1（特徴抽出と学習）のパラダイムシフト**: LightGBM から Transformer（シーケンス処理）または Siamese Neural Network（相対評価）への移行。HodgeRank による非推移的ノイズ除去。GNN による人間関係ネットワーク中心性、トラッキングデータによる幾何学的パス効率性の特徴量追加。
2. **L2（確率統合）の非線形化とバイアス排除**: MLP による非線形統合。一般化 Plackett-Luce モデルの導入によるエキゾチックベットの精緻な確率計算。
3. **L3（購入戦略）の数学的最適化**: argmax の廃止、FLB 回避のオッズ上限フィルター、縮小推定量による過大評価補正、KKT 条件によるマルチアウトカム・ケリー、リスク制約付きフラクショナル・ケリー。

## 12. Part II と Part I の整合性・優先順位への統合

Part II は全体として**「モデルアーキテクチャを刷新すれば α=0 を突破できる」という仮説**に立つが、本プロジェクトがこれまで実測してきた事実（指南書 §6、8手法）は「**入力情報源が同じであれば、モデルの表現力を上げても市場を超えない**」ことを繰り返し示している。したがって Part II の項目は次の基準で仕分けて優先順位を判断すべきである。

| Part II 項目 | 情報源は新規か | Part I との関係 | 推奨する扱い |
|---|---|---|---|
| 7.1 Transformer | 否（既存特徴量のまま） | §6 の変奏（モデル形式変更） | 低優先。P1 が有意セグメントを発見した場合のみ、そのセグメント限定でモデル形式変更の価値を再検討 |
| 7.2 Siamese Network | 否 | 同上 | 同上 |
| 7.3 HodgeRank | 否（前処理） | 前処理改善 | JRA でのペアワイズ密度確認が先決。低〜中優先の調査候補 |
| 8.1 GNN | 否（既存関係性の再表現） | §6 #1 の変奏 | **αゲート必須**。通常の特徴量追加と同列に扱う |
| 8.2 トラッキングデータ | **是**（JV-Link に存在しない） | P3 と同種 | **却下確定**（2026-07-09、JV-Link全38レコード種別を確認済み。該当データなし。専用ベンダー契約が別途必要） |
| 8.3 Benter残差・レーティング | 否 | §6 #1 の変奏 | **αゲート必須** |
| 9.1 非線形融合（MLP） | 否（市場情報の使い方の変更） | **P1 と同一仮説の別実装** | P1 の結果を見てから判断 |
| 9.2 一般化 Plackett-Luce | 否（確率算出方式の変更） | P4 に近い「市場内部の整合性」軸 | Stern/Harville との直接比較で検証可能。中優先 |
| 10.1 FLB フィルター | 否（購入ロジックの改善） | 既存 `max_odds` の拡張 | 低コストで試行可能だが、エッジ創出ではなく分散削減である点に注意 |
| 10.2 縮小推定量 | 否 | winner's curse 対策の高度化 | **中優先**。既存の集中度ゲートより一段進んだ実装候補 |
| 10.3/10.4 マルチアウトカム・ケリー等 | 否（資金管理の最適化） | P5 と同じ「エッジ確立後」 | 低優先。正の EV が1つも無い現状では時期尚早 |

**結論**: Part II のうち、本プロジェクトの制約（L1 市場情報禁止・Rule 3・αゲート）と矛盾せず、かつ現在のデータで即座に検証可能なのは **9.2（Plackett-Luce による券種確率の再計算）** と **10.2（縮小推定量による winner's curse 対策）** の2点であり、これらは Part I の優先順位（P1 → P2 → P4 → P5）に対して**並行して低コストで試せる補助的候補**として追加してよい。7.1/7.2/8.1/8.3 のような大規模アーキテクチャ刷新は、Part I の実測結果（特に P1 セグメント検定の結果）が出るまで着手を保留することを推奨する。**8.2（トラッキングデータ）は却下確定**（JV-Link に存在せず、専用ベンダー契約が別途必要）。

### 補足: P3・8.2 の再検証結果（2026-07-09）

Part II 追加後、ユーザーからの指摘を受けて P3（レース前オッズ時系列）と 8.2（トラッキングデータ）を `docs/JV-Data.md` に照らして再検証し、**両方とも却下**した。

- **8.2 トラッキングデータ**: JV-Link の全38レコード種別を確認したが、GPS/RFID等の走行軌跡データは1つも存在しない。JRA-VAN とは別の専用ベンダー契約が必要であり、現行データソースの範囲では実現不可能と確定。
- **P3 レース前オッズ時系列**: `docs/JV-Data.md` §6(2)速報系データに「時系列オッズ（単複枠）0B41」「時系列オッズ（馬連）0B42」が実在することを確認したが、これらは速報系（リアルタイム配信のみ）に分類され、蓄積系（過去日付での再取得が可能な仕組み）には含まれない。2015〜2025年の既存データへの遡及取得は不可能で、取得できるのは購読開始後に行われる新規レースの分のみ。意味のあるサンプル数が貯まるまで数週間〜数ヶ月を要し、既存の fold2 OOS 枠組みにも統合できないため、「安価な次の一手」の基準を満たさず却下。将来的に前向きデータ収集への投資をユーザーが決断する場合は、本提案書とは独立した意思決定として再提案する。

この再検証により、**優先順位は P1 → P2 → P4 → P5 に確定**する（P3 除外）。
