# track_bias_daily — 馬場バイアス日次推定（Phase A2）

## 目的

`docs/specs/2026-07-12-alpha-recovery-data-expansion-spec.md` Phase A2の実装。
「同日・同場の先行レースの枠番/通過順位別の着順残差」から、その日その競馬場で
内外・前後どちらが有利かを動的に推定し、L1特徴量として追加する。
Phase A（クッション値・含水率、公式発表値）は不合格だったが、これは**実際のレース結果から
逆算したバイアス**であり、情報の質が異なる（公式測定値の限界を、結果から直接推定することで
補う）。ユーザー指示により評価はTop-1/NDCG/Spearman（的中率）のみ、ROI/αは対象外。

## 特徴量設計（2種、いずれもレース内で馬ごとに異なる値を持つ＝静的フラグの死重問題を回避）

### 1. `post_bias_today`（内外バイアス）
同日・同場で当該レースより**前**に行われたレースの結果から、
枠番3分位（内/中/外）ごとの平均着順残差（3分位平均 − 全体平均）を計算し、
当該馬の枠番3分位に対応する値（符号反転、正=有利）を付与する。

### 2. `pace_bias_x_style`（前後バイアス×先行傾向の交互作用）
同日・同場の先行レースにおける「4角通過順位が前半だった馬」と「後半だった馬」の
平均着順差から、その日のペース傾向（先行有利/差し有利）を推定し、
既存の `hist_front_running_pref`（馬個体の先行傾向、shift済み過去走ベース）と掛け合わせる。
前回セッションの学び（静的フラグは死重、交互作用項が必要）を踏まえた設計。

## リーク対策

- 同日・同場・**当該レースより前のrace_num**のみ使用（race_idの末尾2桁=race_num、
  発走順を反映）。当日の最初の1〜2レースは先行データなしのためNaN許容。
- 使用する `corner_4`（4角通過順位）・`finish_rank` はいずれも**先行レース**の結果であり、
  当該レースの結果ではないため後出し情報ではない（`hist_*` 系の「過去走」を「当日の先行レース」
  に置き換えた設計と同型）。
- 十分なサンプル（先行レース2走以上）がない場合はNaNのままLightGBMに渡す。

## プロトコル

- ベース: v39_course_slim（106列）+ 新2列 = 108列
- fold2 OOSで先にスクリーニング → 有望なら3fold + TEST正式評価（Rule 3準拠、1回のみ）
